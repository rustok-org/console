# Console v2 — Фаза 1: визуальный редизайн approver'а

> Управляющий план: `meta/docs/CONSOLE-V2-PLAN.md` §Фаза 1 + §Инварианты.
> Дизайн-референс: артефакт https://claude.ai/code/artifact/9e435945-33c2-4f26-acad-d4c32bb3b176
> Протокол НЕ трогаем: `docs/APPROVER-PROTOCOL.md`. Репо `console` (public, MIT), ratatui 0.30.

## Цель

Сделать approver-консоль **красивой и брендированной** — палитра сайта, framed-панели, человеческие
суммы, тематизированная карточка одобрения — **без изменения протокола, модели и поведения
безопасности**. Это чистый визуальный слой поверх текущей one-shot-консоли; резидентность и
дашборд — Фазы 2-3, вне этого круга.

> **Синхронизация с реальностью (по ходу круга, ратифицировано):** три пункта макета в Фазу 1
> НЕ вошли и вынесены — не по осторожности, а по факту: **анимация** (реш. Капитана — «убрать, если
> сложности»); **сокращённые адреса** (регрессия безопасности — address-poisoning, ловит существующий
> verbatim-тест; адреса на карточке ПОЛНЫЕ); **двух-блочный «From → To»** (в approver-протоколе НЕТ
> адреса отправителя — блок «From: 0x…» физически не собрать без read-op → Фаза 2). Что реально едет
> в Фазе 1: палитра + framed-панели + человеческие суммы + declutter карточки, адреса verbatim.

## Скоуп PR — что входит / что ЯВНО не входит

**Входит:**
1. **Палитра** (токены rustok-landing → `ratatui::style::Color::Rgb`): фон `#0B0712`, рамки
   `#9D5CFF`/`#2A1C4D`, текст `#EDE7FF`, семантика — тил `#16E0C3` (approve/executed), янтарь
   `#FFB454` (high-risk), красный `#FF6B6B` (reject/denied). `NO_COLOR` env → отключить цвет (даём
   деградацию до дефолтного терминала).
2. **Framed-панели** — `Block::bordered()` с заголовками и брендовым цветом рамки на всех экранах
   (auth, watch/queue, card, actions).
3. **Человеческие суммы:** нативный `amount_wei` → `0.01 ETH` (18 знаков, обрезка хвостовых нулей).
   ⚠️ Токенные суммы в `decoded_call.amount` — decimals токена консоли неизвестны, «человечить» их
   нельзя без вранья: оставляем как есть, `UNLIMITED` уже спец-обработан. Форматтер применяется
   ТОЛЬКО к верхнему `amount_wei` (нативное значение, 18 dec — известно).
   **КРИТИЧНО (finding /check):** для token-операций нативный `amount_wei` = `0` (фикстура ui.rs:676),
   а реальная сумма — в `decoded_call.amount`. Нельзя выносить «0 ETH» лидом — карточка спрячет
   движение токенов. Правило: **если есть `decoded_call` с суммой — лид карточки = decoded-значение**
   (как есть), а нативный «0 ETH» уходит с первого плана. Форматтер натив→ETH применяется только
   когда натив реально несёт значение (простой send).
4. **Адреса — ПОЛНЫЕ, verbatim** на карточке одобрения (НЕ сокращаем): clear-signing — место, где
   человек сверяет, КОМУ уходят деньги; `0x1234…abcd` спрятал бы address-poisoning-двойника (инвариант
   `AGENTS.md` #1). Сокращение адресов — только для дашборда Фазы 2 (там дисплей, не подпись).
5. **Дев-поля declutter (статически, без клавиши-тоггла):** строка-шум `decoded_call: (none)` для
   простого перевода не рендерится; `raw_data` уже во вторичной области карточки; для контракт-вызова
   decoded-поля остаются видимы полностью. Тоггл-клавиша НЕ добавляется (это app-состояние = поведение,
   как авто-открытие — вне визуального слоя).
6. **Тематизированная карточка** (НЕ двух-блочный From→To — он Фаза 2, нет адреса отправителя):
   человеческая сумма лидом, риск янтарём, получатель акцентом, полные адреса; строки — внутри
   `priority_lines()` (гейт `priority_fields_fit` считает их, источник истины один — ui.rs). Никакого
   счётчика кадров, `render`-сигнатура и `main.rs` НЕ меняются.

**ЯВНО НЕ входит:**
- **Авто-открытие единственного pending** (issue #7 п.2) — УБРАНО из Фазы 1 (решение Капитана

- **Изменение протокола/сокета**, read-ops, резидентный дашборд, баланс/позиции/история — Фазы 2-3.

## Архитектура и чистота (стандарт Капитана — критерий приёмки, не пожелание)

Продакшен-качество, не «работает и ладно». Конкретно:
- **Палитра — один модуль `src/theme.rs`:** семантические имена (`fn approve()`, `fn high_risk()`,
  `fn reject()`, `fn frame()`, `fn ink()`, `fn muted()`), а не сырые `Color::Rgb(…)` по всему ui.rs.
  Смена бренд-цвета — правка в одном месте.
- **Форматирование — отдельный модуль `src/format.rs`:** `wei_to_eth`, `short_addr`, лид-сумма
  карточки (натив vs decoded). Чистые функции, юнит-тестируемые без рендера.
- **Раскладка карточки — отдельная функция** (`render_flow`/`card_layout`), не разбухший
  `render_detail`; `priority_lines` остаётся единственным источником истины для гейта.
- **Идиоматичный Rust/ratatui 0.30:** без `unwrap` на пользовательском пути, без магических чисел
  (размеры/отступы — именованные конст.), `clippy -- -D warnings` чисто, `rustfmt` чисто.
- **Хирургично:** трогаем только отрисовку (ui.rs + новые theme/format); протокол/транспорт/модель одобрения не
  рефакторим попутно. Комментарии — только там, где объясняют ограничение (напр. инвариант гейта),
  не пересказ кода.

## Критерии приёмки — «PR готов, когда…»

1. Все экраны в брендовой палитре; `NO_COLOR=1` даёт читаемую деградацию (тест).
2. Нативная сумма показана как `0.01 ETH`, не `10000000000000000`; токенные суммы не «очеловечены»
   ложно (тест на форматтер: wei→ETH, обрезка нулей, крайние значения 0/1/MAX).
3. Дев-поля свёрнуты для простого send, видимы для контракт-вызова (снапшот-тест обоих).
5. Статичная From→To раскладка **не ломает `priority_fields_fit`** (её строки внутри `priority_lines()`,
   гейт держится: при нехватке места approve заблокирован, баннер TOO SMALL) — снапшот-тест на грани.
8. Деградация цвета: `NO_COLOR=1` → моно (тест); на не-truecolor терминале `Color::Rgb` даунсэмплится
   ratatui — снапшот/пометка, что бренд-фиолетовые остаются читаемы (finding /check).
9. **Scope-guard (finding /check):** в коде Фазы 1 НЕТ свап-карты / `⇄`-ветки (свап отложен — нужен
   core-декод); греп подтверждает только send/high-risk варианты.
6. **Инварианты целы:** консоль не открывает сокет для чтения баланса (протокол не расширен),
   PIN на high-risk, default-deny по дедлайну, exit-коды #7 не изменены, никаких секретов в UI.
7. Гейты зелёные показанным прогоном: `cargo fmt --check`, `cargo clippy -- -D warnings`,
   `cargo test`, `cargo deny check`. Новые тесты — red→green.
